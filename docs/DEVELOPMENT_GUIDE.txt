# JEMBERKOST_DEVELOPMENT_GUIDE.md

## 1. Tujuan Dokumen

Dokumen ini berfungsi sebagai pedoman teknis pengembangan website JemberKost agar seluruh kode yang dibuat memiliki struktur yang konsisten, mudah dipelihara, scalable, dan mengikuti best practice modern untuk Next.js dan Supabase.

Dokumen ini harus menjadi acuan utama saat menggunakan AI seperti Gemini, ChatGPT, Copilot, atau saat melakukan pengembangan secara manual.

---

# 2. Tech Stack Final

## Frontend

* Next.js 15 (App Router)
* TypeScript
* Tailwind CSS
* Shadcn UI

## Backend

* Supabase PostgreSQL

* Supabase Storage

* React Hook Form

* TanStack Query

## Maps

## Deployment

* Vercel

## Version Control

* Git
* GitHub

# 3. Development Principles

## Clean Architecture

Kode harus dipisahkan berdasarkan tanggung jawab:

### UI Layer

```text
components/
```

Berisi komponen visual.

### Feature Layer

```text
features/
```

Berisi business logic.

### Service Layer

```text
services/
```

Berisi komunikasi dengan Supabase.

### Data Layer

```text
supabase
database
storage
```

## Separation of Concerns

Jangan mencampur:

* UI
* Business Logic
* Database Query

dalam satu file.

# 6. Naming Convention

## Folder

Gunakan:

```text
kebab-case
```

Contoh:

```text
kost-card
review-card
owner-sidebar
```

## File

Gunakan:

```text
kebab-case
```

Contoh:

```text
kost-card.tsx
owner-sidebar.tsx
```

## Component

Gunakan:

```tsx
export function KostCard() {}
```

(PascalCase)

## Interface

```tsx
interface KostCardProps {}
```

## Type

```tsx
type UserRole = "user" | "owner" | "admin";
```

## Constants

Gunakan:

```tsx
MAX_IMAGE_SIZE
DEFAULT_PAGE_SIZE
```

(UPPER_SNAKE_CASE)

---

# 7. TypeScript Rules

## Wajib Menggunakan TypeScript

Hindari penggunaan:

```tsx
any
```

kecuali benar-benar diperlukan.

## Gunakan Type

```tsx
type UserRole = "user" | "owner" | "admin";
```

untuk Union Type.

## Gunakan Interface

```tsx
interface UserProfile {}
```

untuk Object Shape.

---

# 8. Component Rules

## Reusable Components

Simpan di:

```text
components/ui
```

Contoh:

* Button
* Input
* Badge
* Card
* Dialog

## Shared Components

Simpan di:

```text
components/shared
```

Contoh:

* Navbar
* Footer
* Pagination

## Feature Components

Simpan di:

```text
features/kost
features/review
features/auth
```

---

# 9. Server Components & Client Components

## Default

Gunakan Server Component.

## Gunakan "use client" hanya jika:

* useState
* useEffect
* Event Handler
* Browser API

dibutuhkan.

## Hindari

Menjadikan seluruh halaman sebagai Client Component.

---

# 10. State Management Rules

## Server State

Gunakan:

```text
TanStack Query
```

## UI State

Gunakan:

```tsx
useState
```

## Jangan Gunakan

* Redux
* MobX

Karena terlalu kompleks untuk kebutuhan proyek.

---

# 11. Supabase Rules

## Semua Query Harus Melalui Service Layer

Contoh:

```text
services/kost.service.ts
```

## Jangan Query Langsung di Component

Hindari:

```tsx
const { data } = await supabase...
```

langsung di halaman.

## Gunakan Pattern

```text
page
↓
service
↓
supabase
```

---

# 12. Authentication Rules

Role:

```text
guest
user
owner
admin
```

## Guest

Akses:

* /
* /kost
* /kost/[slug]
* /about
* /contact

## User

Akses:

* /dashboard

## Owner

Akses:

* /owner

## Admin

Akses:

* /admin

---

# 13. Middleware Rules

### Guest

Tidak dibatasi.

### User

Harus login.

### Owner

Harus login dan role owner.

### Admin

Harus login dan role admin.

---

# 14. Database Rules

Gunakan UUID sebagai Primary Key.

Semua tabel wajib memiliki:

```sql
created_at
updated_at
```

kecuali tabel pivot.

Gunakan Foreign Key untuk seluruh relasi.

# 21. Error Handling Rules

Gunakan:

```tsx

```

untuk seluruh operasi database.

* Success
* Error

---

# 22. Security Rules

## Jangan Pernah

Menyimpan:

* Service Role Key
* Secret Key

di frontend.

## Gunakan

```text
Row Level Security (RLS)
```

pada seluruh tabel Supabase.

## Owner

Hanya boleh mengelola kos miliknya sendiri.

## User

Hanya boleh mengubah data miliknya sendiri.

---

# 23. Performance Rules

Gunakan Server Component sebisa mungkin.

Gunakan Pagination untuk daftar kos.

Gunakan Lazy Loading untuk gambar.

Optimalkan query Supabase agar hanya mengambil data yang diperlukan.
