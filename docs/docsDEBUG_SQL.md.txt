# Debug SQL Reference - JemberKost

## Overview

Dokumen ini berisi kumpulan query SQL yang sering digunakan untuk debugging database JemberKost pada Supabase.

Gunakan query ini ketika mengalami masalah:

* Permission denied
* Row Level Security (RLS)
* Data tidak muncul
* Dashboard menampilkan angka 0
* Insert gagal
* Update gagal
* Role tidak terbaca
* Verifikasi kos bermasalah

---

# 1. Cek Semua Tabel

```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
```

---

# 2. Cek Struktur Semua Tabel

```sql
SELECT
    table_name,
    column_name,
    data_type
FROM information_schema.columns
WHERE table_schema = 'public'
ORDER BY table_name, ordinal_position;
```

---

# 3. Cek Semua Enum

```sql
SELECT
    t.typname AS enum_name,
    e.enumlabel
FROM pg_type t
JOIN pg_enum e
ON t.oid = e.enumtypid
ORDER BY t.typname, e.enumsortorder;
```

---

# 4. Cek Enum Role

```sql
SELECT enumlabel
FROM pg_enum
WHERE enumtypid = (
  SELECT oid
  FROM pg_type
  WHERE typname = 'user_role'
);
```

---

# 5. Cek Enum Status Kos

```sql
SELECT enumlabel
FROM pg_enum
WHERE enumtypid = (
  SELECT oid
  FROM pg_type
  WHERE typname = 'kost_status'
);
```

# 6. Cek Semua RLS Policy

```sql
SELECT
    schemaname,
    tablename,
    policyname,
    permissive,
    roles,
    cmd,
    qual,
    with_check
FROM pg_policies
ORDER BY tablename;
```

# 20. Cek Relasi Foreign Key

```sql
SELECT
    tc.table_name,
    kcu.column_name,
    ccu.table_name AS foreign_table_name,
    ccu.column_name AS foreign_column_name
FROM information_schema.table_constraints tc
JOIN information_schema.key_column_usage kcu
ON tc.constraint_name = kcu.constraint_name
JOIN information_schema.constraint_column_usage ccu
ON ccu.constraint_name = tc.constraint_name
WHERE tc.constraint_type = 'FOREIGN KEY';
```

# 21. Cek Owner dan Kos

```sql
SELECT
    p.full_name,
    p.email,
    k.name
FROM profiles p
JOIN kosts k
ON p.id = k.owner_id;
```

# 22. Cek Kampus Terdekat yang Tersimpan

```sql
SELECT
    k.name,
    c.name AS campus_name
FROM kosts k
LEFT JOIN campuses c
ON k.campus_id = c.id;
```

# Common Errors

## permission denied

Penyebab:

* Tidak ada policy
* RLS aktif
* Role tidak sesuai

---

## row violates row-level security policy

Penyebab:

* auth.uid() tidak sesuai
* owner_id salah
* user_id salah

---

## invalid input value for enum

Penyebab:

* Nilai enum tidak tersedia di database

Contoh:

```text
pending
```

padahal enum hanya:

```text
available
full
```

---

## Dashboard Menampilkan 0

Cek:

1. Data benar-benar ada
2. Policy SELECT tersedia
3. Role user benar
4. Query filter tidak salah
