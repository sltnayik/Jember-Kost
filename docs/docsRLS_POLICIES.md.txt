# Row Level Security (RLS) Policies

## Overview

JemberKost menggunakan Row Level Security (RLS) Supabase untuk membatasi akses data berdasarkan role pengguna.

RLS memastikan:

* User hanya dapat mengakses data miliknya sendiri
* Owner hanya dapat mengelola kos miliknya sendiri
* Admin dapat melihat seluruh data
* Data publik tetap dapat diakses pengunjung

# Security Notes

## Rule 1

Jangan pernah membuat policy:

```sql
USING (true)
```

untuk tabel sensitif seperti:

* profiles
* favorites
* reports
* verification_requests

karena dapat menyebabkan kebocoran data.

---

## Rule 2

Setiap tabel baru wajib memiliki:

* SELECT policy
* INSERT policy
* UPDATE policy
* DELETE policy

jika tabel digunakan oleh aplikasi.

---

## Rule 3

Setelah membuat tabel baru, selalu lakukan pengujian:

* login user
* login owner
* login admin

untuk memastikan policy berjalan sesuai kebutuhan.
